Django 5.x + Django REST Framework
Python 3.12 · DRF 3.15

Le backend REST de DiploChain. DRF fournit le système d'authentification par token pour les universités. Django orchestre les deux modules principaux : certification (émission) et vérification.

  • Traitement en mémoire (InMemoryUploadedFile) — aucun diplôme stocké sur disque
  • Sérialisation et validation des données entrantes avec DRF Serializers
  • Gestion des exceptions API structurée (codes d'erreur normalisés)
  • Logging sécurisé des actions d'émission (audit trail)
Python 3.12DRF 3.15 RGPD-CompliantToken Auth
💧
Watermarking Numérique (OpenCV + Stegano)
OpenCV 4.x · Stegano 0.11

Le tatouage numérique invisible est la clé de la vérification physique. L'identifiant est encodé dans les fréquences spatiales basses du document via la transformée DCT (Discrete Cosine Transform), une technique éprouvée issue du standard JPEG.

  • Redondance ×8 : injection dans 8 zones différentes du document
  • Robustesse testée à 40% de compression JPEG, rotation 30°, et photocopie
  • Le décodage utilise un vote majoritaire : succès si 60% des zones sont lisibles
  • Prétraitement des images à la vérification : CLAHE, homographie, débruitage

📱
Flutter (Application Mobile)
Flutter 3.x · Dart 3.x

L'application mobile cross-platform pour iOS et Android. Les recruteurs et jurys de concours peuvent vérifier n'importe quel diplôme en le photographiant — résultat en 3 secondes.

  • Interface épurée — aucune connaissance technique requise
  • Accès caméra natif pour capture en temps réel
  • Envoi du fichier à l'API REST via multipart/form-data
  • Affichage du verdict avec les métadonnées blockchain (université, date, TX Algorand)
  • Mode hors ligne : mise en cache des vérifications récentes

Blockchain Algorand
Algorand SDK Python 2.x

Algorand est le réseau blockchain retenu pour ses propriétés uniques adaptées aux registres officiels : finalité absolue, absence de forks, faible coût et très haute vitesse.

  • Pure Proof-of-Stake (PPoS) — finalité en 2.8 secondes, sans risque de réorganisation
  • 5 716 TPS maximum en 2024 — scalable pour des millions de diplômes
  • Coût par transaction : 0.001 ALGO ≈ 0.5 FCFA — quasi gratuit pour les universités
  • Utilisé par des gouvernements (République de Marshall Islands, Italie) pour des registres officiels
  • Explorateur public : tout enregistrement est consultable sans compte
PPoS5716 TPS Finalité 2.8s0.001 ALGO/tx

Sécurité Cryptographique
SHA-256 · Ed25519 · .env Vault

La sécurité repose sur deux primitives cryptographiques éprouvées et complémentaires :

  • SHA-256 — Algorithme de hachage (famille SHA-2) produisant une empreinte de 256 bits. Résistant à toutes les attaques connues par collision. Utilisé dans Bitcoin, TLS, et les certificats X.509.
  • Ed25519 — Algorithme de signature à courbe elliptique utilisé par Algorand. Chaque université dispose d'une paire (clé privée / clé publique). La clé privée est stockée dans un fichier .env non commité.
  • Les clés privées ne sont jamais envoyées ou stockées côté serveur DiploChain
  • Modèle Zero-Trust : même un administrateur DiploChain ne peut pas forger de diplôme
SHA-256Ed25519 Zero-Trust.env Vault

Pipeline de Vision par Ordinateur
OpenCV 4.x · Pillow · NumPy

La robustesse du système de vérification face aux conditions réelles (photos obliques, mauvais éclairage) repose sur un pipeline de prétraitement d'image en plusieurs étapes :

  • Détection des contours du document dans la photo (algorithme de Canny)
  • Correction de perspective via transformation homographique (redressement)
  • Amélioration du contraste adaptative (CLAHE — Contrast Limited Adaptive Histogram Equalization)
  • Débruitage gaussien pour réduire le bruit des capteurs photo mobiles
  • Normalisation de la résolution avant extraction du tatouage

Sécurité

Modèle de sécurité Zero-Trust

DiploChain est conçu pour qu'aucune compromission du serveur ne puisse permettre la création de faux diplômes.

🔑
Clés privées hors serveur
Les clés privées des universités ne sont jamais envoyées à DiploChain. Elles restent sur les systèmes de l'établissement, dans un fichier .env sécurisé.
🗂️
Données traitées en mémoire
Aucun diplôme n'est jamais écrit sur le disque du serveur. Traitement 100% RAM — conformité totale avec le RGPD sur les données personnelles.
⛓
Registre décentralisé
Le registre de référence est sur Algorand, pas sur les serveurs de DiploChain. Même si DiploChain disparaît, toutes les transactions restent consultables.
🔏
Signature non reproductible
Sans la clé privée d'un établissement, il est cryptographiquement impossible d'enregistrer un diplôme en son nom. Aucune permission admin ne peut contourner cela.
📋
Audit trail complet
Toutes les actions d'émission sont journalisées avec timestamp, adresse IP et identifiant de l'université. La blockchain elle-même constitue un deuxième audit immutable.
🌐
Vérification publique
N'importe qui dans le monde peut consulter les transactions sur l'explorateur Algorand public — sans compte, sans API key, sans intermédiaire.
